# API Reference — REST Endpoints

The Hestia Labs REST API provides 6 implemented endpoints for device management, secret rotation, and command dispatch. The API is **Cloud-facing** and runs on Cloud Standard deployments or Local AI Nodes.

**Protocol:** JSON over HTTPS with Bearer JWT authentication  
**Base URL:** `https://api.hestia.local/v1` (Cloud/Local AI Node)  
**Port:** 443 (HTTPS)  
**Encoding:** UTF-8 JSON  
**Version:** v1 (stable)

---

## Implementation Status

| Category | Count | Status |
|---|---|---|
| **Implemented** | 6 | Complete and stable |
| **Planned** | 4 | Specification-defined, not yet built |

### 6 Implemented Endpoints

| # | Method | Path | Min Role | Purpose |
|---|---|---|---|---|
| 1 | `POST` | `/device/register` | admin | Register device in home |
| 2 | `GET` | `/devices` | viewer | List all home devices |
| 3 | `POST` | `/device/:id/rotate-secret` | admin | Rotate device HMAC secret |
| 4 | `GET` | `/device/:id/key-history` | admin | View secret rotation audit trail |
| 5 | `POST` | `/device/:id/revoke` | admin | Disable device access |
| 6 | `POST` | `/device/:id/command` | operator | Send command to device |

### 4 Planned Services (Not Yet Implemented)

- **Intent Service API** `[Planned]` — HX47 Planner formally submits signed intents (Ed25519)
- **Manifest Service API** `[Planned]` — Devices register capabilities at runtime
- **Provisioning API** `[Planned]` — QR-token device claim flow
- **Edge Service REST** `[Planned]` — Route decisions and policy evaluation exposed

## Authentication

All endpoints require Bearer JWT token:

```
Authorization: Bearer <JWT_TOKEN>
```

**Token Format:** JWT with `aud: "hestia"`, `sub: <user-id>`, `role: admin|operator|viewer`

**Getting a Token:** Contact your Hestia Labs configured auth service (Keycloak, etc.)

**Signature Layer (Not Yet Implemented):** The specification requires Ed25519 Planner-signed intents for the Intent Service API. This is **Planned — Not Implemented**. Currently, commands over REST use JWT auth only. Devices verify signatures over MQTT/mTLS.

---

## Request Format

All requests follow this structure:

```json
{
  "action": "turn_on",
  "params": {
    "brightness": 75
  }
}
```

- **Content-Type:** `application/json`
- **Charset:** UTF-8
- **Size limit:** 16 KB max payload

---

## Response Format

All responses follow this structure:

**Success (2xx):**
```json
{
  "status": "success",
  "data": {
    "device_id": "helix-kitchen-01",
    "action": "turn_on",
    "result": "executed"
  }
}
```

**Error (4xx/5xx):**
```json
{
  "status": "error",
  "error_code": "ERR_DEVICE_NOT_REACHABLE",
  "message": "Device is offline or unreachable",
  "http_status": 503,
  "timestamp": "2026-02-27T10:34:56Z"
}
```

## Endpoint Details

### 1. Register Device

Register a new device into the home.

```
POST /device/register
```

**Role Required:** admin  
**Feature Flag:** `Flags.REGISTRATION`

**Request:**
```json
{
  "device_id": "helix-kitchen-01",
  "device_type": "helix_node",
  "friendly_name": "Kitchen Light"
}
```

**Response (201 Created):**
```json
{
  "status": "success",
  "data": {
    "device_id": "helix-kitchen-01",
    "secret": "hmac-sha256-secret-hex-64-chars",
    "created_at": "2026-02-27T10:34:56Z"
  }
}
```

### 3. Rotate Secret

Rotate a device's HMAC secret for security.

```
POST /device/:id/rotate-secret
```

**Role Required:** admin  
**Feature Flag:** `Flags.SECRET_ROTATION`

**Request:**
```json
{
  "reason": "security_audit"
}
```

**Response (200 OK):**
```json
{
  "status": "success",
  "data": {
    "device_id": "helix-kitchen-01",
    "new_secret": "new-hmac-sha256-secret-hex-64-chars",
    "old_secret_revoked_at": "2026-02-27T10:35:00Z",
    "message": "Secret rotated. Update device firmware immediately."
  }
}
```

### 5. Revoke Device

Revoke device access. The device is disabled and cannot receive commands.

```
POST /device/:id/revoke
```

**Role Required:** admin

**Request:**
```json
{
  "reason": "device_removed"
}
```

**Response (200 OK):**
```json
{
  "status": "success",
  "data": {
    "device_id": "helix-kitchen-01",
    "revoked_at": "2026-02-27T10:36:00Z",
    "reason": "device_removed",
    "revoked_by": "admin@example.com",
    "message": "Device access revoked. Device will be removed from home."
  }
```

---

### 6. Send Command

Send a command to a device. Commands are routed through the MQTT router to the device.

```
POST /device/:id/command
```

**Role Required:** operator

**Request:**
```json
{
  "action": "turn_on",
  "params": {
    "brightness": 75,
    "color_temperature": 3200
  }
}
```

**Response (202 Accepted):**
```json
{
  "status": "success",
  "data": {
    "device_id": "helix-kitchen-01",
    "intent_id": "intent-550e8400-e29b-41d4-a716",
    "status": "dispatched",
    "estimated_execution_time": "200ms"
  }
}
```

**Error (503 Service Unavailable):**
```json
{
  "status": "error",
  "error_code": "ERR_DEVICE_NOT_REACHABLE",
  "message": "Device is offline or MQTT connection failed",
  "device_id": "helix-kitchen-01"
}
```

## Data Flow

```
Client (REST, JSON/JWT)
    ↓
POST /device/:id/command
    ↓
CommandEngine (validates action, params)
    ↓
MQTT Router (internal)
    ↓
Device (MQTT mTLS, binary HxTP)
    ↓
Device verifies signature (HMAC-SHA256)
    ↓
Device executes action within capability bounds
    ↓
Device reports result to Edge Service
    ↓
Result returned to client
```

---

## Transport Layers

### REST API (Client-Facing)

- **Protocol:** HTTPS/TLS 1.3
- **Format:** JSON
- **Auth:** Bearer JWT
- **Endpoints:** 6 implemented

### MQTT (Device-Facing)

- **Protocol:** MQTT over mTLS
- **Format:** Binary HxTP (not JSON)
- **Auth:** Device HMAC secret + mTLS certificate
- **Router:** Internal MQTT Router dispatches to devices

**This reference covers REST only.** See [HxTP Protocol](/protocol/hxtp-protocol) for binary device protocol details.

## Quick Reference

### Example: Register and Control a Device

```bash
# 1. Register device
curl -X POST https://api.hestia.local/v1/device/register \
  -H "Authorization: Bearer $JWT" \
  -H "Content-Type: application/json" \
  -d '{"device_id": "helix-01", "device_type": "helix_node"}'

# 2. Send command
curl -X POST https://api.hestia.local/v1/device/helix-01/command \
  -H "Authorization: Bearer $JWT" \
  -H "Content-Type: application/json" \
  -d '{"action": "turn_on", "params": {"brightness": 100}}'
```
